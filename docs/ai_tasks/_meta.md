## Workflow

1. **Read** → Open the task file (basic or advanced)
2. **Execute** → Use minimum code approach and use ramda.js for utility functions
3. **Verify** → Test the implementation manually
4. **Document** → Update the tutorial file if specified, but DO NOT create new docs

## Rules

- Fit existing structure and architecture
- One task at a time
- Test before proceeding
- Update tutorials when indicated
